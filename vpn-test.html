💻 Command-Line Tests

Check Public IP

# Linux / macOS
curl ifconfig.me
curl ipinfo.io
curl api.ipify.org

# Windows PowerShell
Invoke-RestMethod ifconfig.me
(Invoke-RestMethod ipinfo.io).ip

DNS Resolution Check

# Linux / macOS
dig google.com
nslookup google.com
resolvectl status        # shows which DNS is being used

# Windows
nslookup google.com
ipconfig /all            # look for DNS Servers line

Check VPN Interface

# WireGuard
sudo wg show             # shows handshake + data + peers
ip addr show wg0         # Linux: interface info
ifconfig utun4           # macOS: WireGuard uses utun* interfaces

# OpenVPN
ip addr show tun0
ps aux | grep openvpn

Traceroute (see the path your packets take)

# Linux / macOS
traceroute google.com
mtr google.com           # better interactive version

# Windows
tracert google.com

Ping the VPN Server

# Replace with your server's IP
ping -c 10 vpn-server.com
ping -c 10 10.88.88.1    # WireGuard internal IP

📱 The Ultimate Phone Test

This is the definitive test for your home VPN server:
  1. Turn OFF WiFi on your phone → use 4G/5G mobile data
  2. Open WireGuard app → toggle VPN ON
  3. Open browser → type 192.168.1.254
  4. If Freebox OS loads → VPN works! You're reaching your home router through mobile data via the encrypted tunnel.
  5. Disconnect VPN → try again → should fail (proves the VPN was doing the work)
