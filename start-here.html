PHASE 1 Set Up Your VPN Server

Choose your protocol based on your Freebox model:

⚡ WireGuard

Freebox One, Delta, Pop, Ultra

Faster, modern, QR code import, recommended

🔐 OpenVPN

Freebox Revolution, Mini 4K (+ all models)

Battle-tested, widely supported

⚡ WireGuard Server Setup (Delta/Pop/Ultra/One)

1
Open Freebox OS
⏱️ 1 minute
  1. Connect to your Freebox WiFi or Ethernet
  2. Open browser → type http://mafreebox.freebox.fr
  3. Log in with your admin password
  4. If first time: confirm on the Freebox LCD screen
Can't access? Try 192.168.1.254 instead. Make sure you're on the Freebox network, not another WiFi.
2
Enable WireGuard Server
⏱️ 2 minutes
  1. Navigate to Paramètres → VPN → Serveur VPN (Settings → VPN → VPN Server)
  2. Click the WireGuard tab
  3. Toggle "Activer" (Enable) → ON
  4. Port: keep 51820 (default) unless you have a reason to change
  5. Click "Appliquer" (Apply)
💡 Port forwarding is usually automatic on newer Freebox models. No extra router config needed!
3
Create Client Peers
⏱️ 2 minutes per device

Create one peer for each device you want to connect:

  1. Click "Ajouter un pair" (Add peer)
  2. Name: phone, laptop, work-pc, etc.
  3. The Freebox auto-generates all keys — you don't need to do anything
  4. Click "Sauvegarder" (Save)
  5. Repeat for each device
4
Download Configs / QR Codes
⏱️ 1 minute

For each peer you created:

  • Click the 📥 download icon → saves a .conf file
  • Or click the QR code icon → scan directly with phone (fastest!)
⚠️ Keep these files safe! Anyone with the config file can connect to your home network.

🔐 OpenVPN Server Setup (Freebox Revolution)

1
Open Freebox OS
⏱️ 1 minute
  1. Connect to your Freebox Revolution WiFi or Ethernet
  2. Open browser → http://mafreebox.freebox.fr (or 192.168.1.254)
  3. Log in with admin password
  4. Navigate to Paramètres → Mode Avancé → VPN
2
Enable OpenVPN Server
⏱️ 2 minutes
  1. Click "Serveur VPN" (VPN Server) tab
  2. Select "OpenVPN"
  3. Toggle "Activer" (Enable) → ON
  4. Port: keep 1194 (default)
  5. Protocol: UDP (recommended — faster than TCP)
  6. Click "Appliquer" (Apply)
3
Create VPN Users
⏱️ 2 minutes per user
  1. Go to "Utilisateurs" (Users) tab
  2. Click "Ajouter un utilisateur" (Add user)
  3. Login: choose a name (e.g., phone, laptop)
  4. Password: set a strong password (12+ characters)
  5. Click "Sauvegarder" (Save)
4
Download Config Files
⏱️ 1 minute
  1. Click the 📥 download icon next to each user
  2. Save the .ovpn file for each device
  3. The file contains: server address, port, certificates, encryption settings
The config uses your Freebox's public IP. If it changes, set up Dynamic DNS: Freebox OS → Paramètres → DynDNS
💡 Server is ready! Now let's connect your devices.

PHASE 2 Connect Your Devices (Clients)

Pick your device and protocol below. Each section is self-contained — follow it from top to bottom.

⚡📱 WireGuard on Phone (iPhone / Android)

🍎 iPhone / iPad

  1. Open App Store
  2. Search "WireGuard" (developer: WireGuard Development Team / Jason A. Donenfeld)
  3. Tap Get → Confirm with Face ID / Touch ID
  4. Open the WireGuard app
  5. QR Code method (fastest!):
    • Tap "+" → "Create from QR code"
    • Point your camera at the QR code shown in Freebox OS
    • Name the tunnel (e.g., "Home")
    • iOS asks "Allow VPN Configurations?" → Tap "Allow"
  6. File method:
    • Send the .conf file to your phone (email, AirDrop, iCloud Drive)
    • Tap the file → "Open in WireGuard?" → Yes
    • Or: in WireGuard app → "+" → "Create from file or archive" → select from Files
  7. Tap the toggle switch next to the tunnel → turns green = connected!
  8. A VPN icon appears in your status bar (🔒 or VPN badge)

🤖 Android

  1. Open Google Play Store
  2. Search "WireGuard" → Install
  3. Open the WireGuard app
  4. QR Code method:
    • Tap "+" (blue button) → "Scan from QR code"
    • Scan the QR from Freebox OS
    • Name it → Save
  5. File method:
    • Tap "+" → "Import from file or archive"
    • Browse to your .conf file → select it
  6. Tap the tunnel → toggle ON
  7. Android asks to set up VPN → tap "OK"
  8. Key icon (🔑) appears in status bar = connected!

⚡💻 WireGuard on Desktop (Windows / macOS)

💻 Windows

  1. Go to wireguard.com/install
  2. Click "Download Windows Installer"
  3. Run the .msi file → Next → Install → Finish
  4. WireGuard opens (also available in system tray)
  5. Click "Import tunnel(s) from file"
  6. Select your .conf file from Freebox
  7. The tunnel appears in the list
  8. Click "Activate"
  9. Status shows Active with green indicator = connected!
  10. To disconnect: click "Deactivate"

🍎 macOS

  1. Open App Store → search "WireGuard" → Get
  2. Or via terminal: brew install wireguard-tools
  3. Open WireGuard app
  4. Click "+" → "Import Tunnel(s) from File..."
  5. Select your .conf file
  6. macOS asks: "WireGuard would like to Add VPN Configurations" → enter your Mac password → "Allow"
  7. Click "Activate"
  8. WireGuard icon appears in menu bar → green dot = connected!

⚡🐧 WireGuard on Linux (Ubuntu / Debian)

# 1. Install WireGuard
sudo apt update && sudo apt install wireguard -y

# 2. Copy your config file
sudo cp ~/Downloads/freebox-phone.conf /etc/wireguard/freebox.conf
sudo chmod 600 /etc/wireguard/freebox.conf

# 3. Connect
sudo wg-quick up freebox

# 4. Verify
sudo wg show          # Shows handshake + data transfer
curl ifconfig.me      # Should show Freebox public IP

# 5. Disconnect
sudo wg-quick down freebox

# 6. Auto-start on boot (optional)
sudo systemctl enable wg-quick@freebox

🔐📱 OpenVPN on Phone (iPhone / Android)

🍎 iPhone / iPad

  1. Open App Store
  2. Search "OpenVPN Connect" (developer: OpenVPN Technologies)
  3. Tap Get → Install
  4. Send the .ovpn file to your phone:
    • Email: Email it to yourself → open on phone → tap attachment → "Open in OpenVPN"
    • AirDrop: Send from Mac → accept on iPhone → opens in OpenVPN
    • Files: Save to iCloud Drive → In OpenVPN: "+" → "Upload File" → select from Files
  5. OpenVPN shows the profile → tap "Add"
  6. Enter the username and password you created in Freebox OS
  7. Tap the toggle switch to connect
  8. iOS asks "Allow VPN?" → tap "Allow"
  9. Status: Connected → VPN icon appears in status bar

🤖 Android

  1. Open Play Store → search "OpenVPN Connect" → Install
  2. Open the app
  3. Tap "+" → "Upload File"
  4. Browse to your .ovpn file (Downloads folder, email attachment, or cloud storage)
  5. Tap "Import"
  6. Enter username and password
  7. Tap "Connect"
  8. Android asks to allow VPN → tap "OK"
  9. Key icon (🔑) appears in status bar = connected!

🔐💻 OpenVPN on Desktop (Windows / macOS)

💻 Windows

  1. Download OpenVPN Connect from openvpn.net/client
  2. Run the installer → Next → Install → Finish
  3. Windows may ask to install a TAP adapter → click "Yes" (required!)
  4. Open OpenVPN Connect
  5. Click "+" → "Upload File"
  6. Select your .ovpn file
  7. Enter username and password from Freebox
  8. Click "Connect"
  9. System tray icon turns green = connected!
⚠️ If connection fails, try Run as Administrator: right-click OpenVPN Connect → "Run as administrator"

🍎 macOS

  1. Download OpenVPN Connect from openvpn.net/client
  2. Or install Tunnelblick (free, open source): tunnelblick.net
  3. Open the app → drag & drop your .ovpn file into the window
  4. Or: click "+" → "Import from file" → select file
  5. Enter credentials → click "Connect"
  6. macOS asks for your admin password → enter it
  7. VPN icon appears in menu bar = connected!

🔐🐧 OpenVPN on Linux (Ubuntu / Debian)

# 1. Install OpenVPN
sudo apt update && sudo apt install openvpn -y

# 2. Connect (foreground — see output)
sudo openvpn --config ~/Downloads/freebox.ovpn
# Enter username and password when prompted
# Ctrl+C to disconnect

# 3. Connect (background — daemon mode)
sudo openvpn --config ~/Downloads/freebox.ovpn --daemon
# To disconnect: sudo killall openvpn

# 4. Auto-start on boot
sudo cp ~/Downloads/freebox.ovpn /etc/openvpn/client/freebox.conf
# Create credentials file:
echo -e "your-username\nyour-password" | sudo tee /etc/openvpn/client/freebox-auth.txt
sudo chmod 600 /etc/openvpn/client/freebox-auth.txt
# Add "auth-user-pass freebox-auth.txt" to freebox.conf
sudo systemctl enable openvpn-client@freebox

# 5. Verify
curl ifconfig.me      # Should show Freebox public IP

PHASE 3 Test Everything Works

You MUST verify your VPN is actually working. Here are 5 tests to run from every connected device.

💻 Test from Desktop (Windows / macOS / Linux)

🕵️
Test 1: IP Check
Open browser → whatismyip.com
Should show your Freebox's public IP, NOT your current network's IP
📍
Test 2: Location
Same site shows your city/country
Should match your Freebox's location (your home city)
🔒
Test 3: DNS Leak
Open dnsleaktest.com → Extended Test
DNS servers should NOT show your current ISP
⚡
Test 4: Speed
Open fast.com
Small drop is normal: WireGuard ~5%, OpenVPN ~15-30%

🔧 Command-Line Tests (Advanced)

# Check your public IP (should be Freebox IP)
curl ifconfig.me

# Check DNS resolution (should NOT be your current ISP)
nslookup google.com

# Ping your home network (should work through VPN)
ping 192.168.1.1          # Your Freebox
ping 192.168.1.100        # A device at home (NAS, Pi, etc.)

# Check VPN interface is active
# WireGuard:
sudo wg show              # Shows handshake time + data transferred
# OpenVPN:
ip addr show tun0         # Shows the VPN tunnel interface

# Trace the route (should go through VPN tunnel)
traceroute google.com     # First hop should be VPN gateway

📱 Test from Phone (iPhone / Android)

⚠️ IMPORTANT: Disconnect from your home WiFi first! Use mobile data (4G/5G) or another WiFi network. Testing while on your Freebox's own WiFi proves nothing — you need to be OUTSIDE your home network.
📶
Step 1: Disconnect WiFi
Turn OFF WiFi on your phone
Use mobile data (4G/5G)
🔒
Step 2: Connect VPN
Open WireGuard/OpenVPN app
Toggle your tunnel ON
🕵️

Open Safari/Chrome → whatismyip.com
Should show Freebox IP, not mobile carrier IP
🏠
Step 4: Access Home
Try 192.168.1.254 in browser
Should open Freebox OS = you're home!

📱 The Ultimate Phone Test

  1. Turn OFF WiFi → use 4G
  2. Connect VPN
  3. Open browser → type 192.168.1.254
  4. If you see Freebox OS → VPN works perfectly! You're accessing your home router from mobile data through an encrypted tunnel.
  5. Try accessing a shared folder on your home computer or NAS
  6. Disconnect VPN → try 192.168.1.254 again → should fail. This proves the VPN was doing the work.

🔬 Advanced Tests

🔒 WebRTC Leak Test

Even with VPN, your browser might leak your real IP via WebRTC.

  1. Open browserleaks.com/webrtc
  2. Check "Public IP Address" — should show VPN IP, not your real IP
  3. If it shows your real IP: disable WebRTC in your browser
  4. Firefox: type about:config → search media.peerconnection.enabled → set to false

🌐 IPv6 Leak Test

Some VPNs only tunnel IPv4, leaking your IPv6 address.

  1. Open test-ipv6.com
  2. If it shows an IPv6 address that's not your VPN's → you have a leak
  3. Fix: disable IPv6 on your device while using VPN, or ensure your VPN handles IPv6

📡 Kill Switch Test

Test what happens when VPN drops unexpectedly.

  1. Connect to VPN
  2. Open whatismyip.com — note VPN IP
  3. Quickly disconnect VPN (toggle OFF)
  4. Immediately refresh whatismyip.com
  5. If your real IP flashes even for a second → no kill switch protection
  6. WireGuard: add PostDown = iptables -I OUTPUT ! -o %i -m mark ! --mark 51820 -j DROP for a basic kill switch

🏓 Latency Test

Measure the VPN overhead in milliseconds.

# Without VPN:
ping -c 10 google.com | tail -1
# Example: avg 15ms

# With VPN:
ping -c 10 google.com | tail -1
# Example: avg 25ms

# Difference = VPN overhead
# WireGuard: typically 5-15ms extra
# OpenVPN: typically 20-50ms extra
