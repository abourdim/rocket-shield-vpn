❓ Frequently Asked Questions

General

What is Rocket Shield VPN?

An educational web app that teaches kids (ages 6-15) about VPNs and cybersecurity through a space-themed interactive experience. It includes a VPN simulator, learning cards, quizzes, and real VPN setup guides.

Is this a real VPN?

The web app itself is a simulator — it teaches how VPNs work but doesn't create real VPN connections. However, the project includes launch scripts that can set up real WireGuard/OpenVPN client and server configurations.

Who is it for?

Students aged 6-15, teachers, parents, coding clubs, and anyone curious about VPN technology. The launch scripts are for more advanced users who want to deploy real VPN infrastructure.

Is it free?

Yes! It's 100% free and open source under the MIT License.

What languages are supported?

English, French, and Arabic (with full RTL support). The language can be switched instantly from the header dropdown.

Does it work offline?

Yes! Once the page loads, everything works offline. The only external dependency is Google Fonts, which falls back to system fonts if unavailable.

Web App

How do I run the app?

Three ways: (1) Double-click index.html, (2) Run bash rocket-shield.sh and select option 1, (3) docker-compose up -d.

How do I change the theme?

Use the theme dropdown in the top-right corner. Choose from Space (dark navy), Desert (warm orange), Synthwave (neon purple), or Paper (light mode).

Are there easter eggs?

Yes! Click ByteBot 5 times for a secret dance. Enter the Konami Code (↑↑↓↓←→←→BA) for rainbow mode. Complete all checklist items for confetti!

Does it work on mobile?

Yes, it's fully responsive. Works on phones, tablets, and any screen size.

VPN Client

Which VPN providers work?

Any provider that offers WireGuard (.conf) or OpenVPN (.ovpn) config files: NordVPN, ProtonVPN, Mullvad, Surfshark, Private Internet Access, and many others.

💡 We recommend WireGuard for most users. Use OpenVPN if you need TCP mode for restrictive networks.

Does a VPN slow my internet?

Slightly, due to encryption overhead. WireGuard typically reduces speed by only 5-10%. OpenVPN can be 15-30% slower. Choose a server close to your location for best speeds.

VPN Server

Can I host a VPN on Raspberry Pi?

Yes! WireGuard runs excellently on Pi 3B+, Pi 4, and Pi 5. The launch script auto-detects Pi hardware and optimizes settings. See the Raspberry Pi Guide.

What port do I need to forward?

WireGuard: 51820/UDP (default). OpenVPN: 1194/UDP or 443/TCP. Forward these on your router/Freebox to your server's local IP.

How many clients can connect?

WireGuard: theoretically unlimited, practically 50-100+ on a Pi 4. Each client needs its own key pair and config file.

Freebox

Which Freebox models support VPN?

Freebox Revolution, Mini 4K, One, Delta, Pop, and Ultra all support VPN. Newer models (Delta, Pop, Ultra) have WireGuard support. All support OpenVPN.

How do I access Freebox OS?

Open a browser and go to mafreebox.freebox.fr while connected to your Freebox network. You'll need the admin password (set it up on first access).

Troubleshooting

VPN won't connect

  • Check your config file is correct (no typos in keys)
  • Verify the server endpoint and port are reachable
  • Check firewall rules on both sides
  • Try restarting the VPN service

DNS leaks detected

  • Add DNS = 1.1.1.1, 8.8.8.8 to your WireGuard [Interface] section
  • For OpenVPN, ensure push "dhcp-option DNS 1.1.1.1" is in server config
  • Disable WebRTC in your browser

Slow VPN speed

  • Switch to WireGuard (faster than OpenVPN)
  • Choose a server closer to your location
  • Check MTU settings (try MTU = 1380)
  • On Pi: ensure you're not CPU-bound (check with htop)

How to check if VPN is working

  1. Visit whatismyip.com — should show VPN server IP
  2. Visit dnsleaktest.com — run extended test
  3. Run wg show to see active WireGuard connections
  4. Use the launch script option 4 (Status & Diagnostics)

⚡ WireGuard Reference

Overview

WireGuard is a modern, fast, and simple VPN protocol. With only ~4,000 lines of code (vs OpenVPN's ~100,000), it's easier to audit, faster to connect, and uses state-of-the-art cryptography.

FeatureDetails
EncryptionChaCha20-Poly1305
Key ExchangeCurve25519 (ECDH)
HashingBLAKE2s
TransportUDP only
HandshakeNoise Protocol Framework (1-RTT)

How It Works

WireGuard uses Cryptokey Routing: each peer has a public key and a list of allowed IP addresses. When a packet arrives, WireGuard checks which peer's allowed IPs match the destination, encrypts it with that peer's public key, and sends it to that peer's endpoint.

Key Generation

# Generate private key
wg genkey > privatekey

# Derive public key from private key
cat privatekey | wg pubkey > publickey

# Generate preshared key (optional, post-quantum security)
wg genpsk > presharedkey

# One-liner: generate key pair
wg genkey | tee privatekey | wg pubkey > publickey

Example: Server Config

# /etc/wireguard/wg0.conf — Server
[Interface]
Address = 10.66.66.1/24           # Server's VPN IP
ListenPort = 51820                # UDP port
PrivateKey = SERVER_PRIVATE_KEY   # From wg genkey

# NAT and forwarding rules
PostUp = iptables -A FORWARD -i %i -j ACCEPT; iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
PostDown = iptables -D FORWARD -i %i -j ACCEPT; iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE

# Client 1: Phone
[Peer]
PublicKey = CLIENT1_PUBLIC_KEY
AllowedIPs = 10.66.66.2/32

# Client 2: Laptop
[Peer]
PublicKey = CLIENT2_PUBLIC_KEY
AllowedIPs = 10.66.66.3/32

Example: Client Config

# client.conf — Phone/Laptop
[Interface]
Address = 10.66.66.2/24
PrivateKey = CLIENT_PRIVATE_KEY
DNS = 1.1.1.1, 8.8.8.8

[Peer]
PublicKey = SERVER_PUBLIC_KEY
Endpoint = your-server-ip:51820
AllowedIPs = 0.0.0.0/0           # Route ALL traffic through VPN
PersistentKeepalive = 25         # Keep connection alive behind NAT

🔴 Opera Browser — FREE Built-in VPN

The easiest way to start. Open Opera → Settings → Privacy → Enable VPN → Toggle ON. A blue VPN badge appears in the address bar. Choose Americas, Europe, or Asia. No account needed!

Limitation: Protects browser traffic only, not your whole device.

Protects the entire device. Built on WireGuard. Includes firewall. Download Brave → Settings → Brave VPN.

Separate app by the Firefox team. Uses WireGuard. Available on Windows, macOS, Linux, Android, iOS. Download from vpn.mozilla.org.

🍎 iCloud Private Relay (iCloud+ $0.99/mo)

Built into Safari on iPhone/iPad/Mac. Not a full VPN — hides your IP and encrypts Safari traffic only. Two-hop architecture. Enable: Settings → iCloud → Private Relay → ON.

🤖 Android Built-in VPN

Settings → Network & Internet → VPN → "+" → Enter server details (IKEv2, L2TP, PPTP) → Save → Connect. No app needed, but requires server info from a provider.

📱 iOS Built-in VPN

Settings → General → VPN & Device Management → Add VPN Configuration → Choose IKEv2/IPSec/L2TP → Enter server details → Done → Toggle ON.

Part 4: VPN Server — WireGuard on Linux/Pi

  1. Install: sudo apt install wireguard
  2. Generate keys: wg genkey | tee server_private | wg pubkey > server_public
  3. Create config: Write /etc/wireguard/wg0.conf with server settings
  4. Enable forwarding: sudo sysctl -w net.ipv4.ip_forward=1
  5. Open firewall: sudo ufw allow 51820/udp
  6. Start: sudo wg-quick up wg0
  7. Create client configs with the launch script (option 3 → Add Client)
💡 Or just run bash rocket-shield.sh and select option 3 — it does all of this automatically!

Part 5: VPN Server on Freebox

  1. Open mafreebox.freebox.fr in your browser
  2. Go to Settings → VPN → VPN Server
  3. Enable WireGuard or OpenVPN server
  4. Create a user and download the config file

See the full Freebox Guide for details.

Step 1: Download your config file

In Freebox OS (or from your server), download the client configuration file: .conf for WireGuard or .ovpn for OpenVPN.

Step 3: Import the config file

📱 Phone (iOS / Android)

Send the config file to your phone (email attachment, AirDrop, USB, or cloud storage). Tap the file → it will prompt "Open in WireGuard?" → Tap Yes → Tap "Allow" to add the VPN tunnel.

Alternative: In the WireGuard app, tap "+" → "Create from file or archive" → select the .conf file.

QR code method: If you used the launch script, it generates a QR code. In WireGuard app, tap "+" → "Create from QR code" → scan it.

💻 PC / Mac (WireGuard)

Open the WireGuard app → click "Import tunnel(s) from file" (or the "+" button on macOS) → navigate to your .conf file → select it → the tunnel appears in the list.

💻 PC / Mac (OpenVPN)

Open OpenVPN Connect → click "+" → "Import from file" → select your .ovpn file → enter credentials if prompted → click "Connect".

🐧 Linux (command line)

WireGuard: sudo cp your-config.conf /etc/wireguard/wg0.conf && sudo wg-quick up wg0

OpenVPN: sudo openvpn --config your-config.ovpn

Step 4: Connect!

Open the app → tap/click the toggle switch next to your tunnel name → it turns green = connected! When you're away from home, just toggle ON to access your home network securely.

VPN Client Mode

Routes ALL home traffic through an external VPN provider.

  1. Go to VPN → VPN Client → Add Connection
  2. Select protocol (OpenVPN or WireGuard)
  3. Upload your provider's config file (.ovpn or .conf)
  4. Enter credentials if required
  5. Save and toggle ON

VPN Server Mode

Host your own VPN to access home network remotely.

  1. Go to VPN → VPN Server
  2. Choose protocol and enable
  3. Create a VPN user with strong password
  4. Download the generated client config file
  5. Import config on your phone/laptop
Port forwarding is automatic on Freebox — no manual setup needed for the VPN server port.

  1. Open mafreebox.freebox.fr
  2. Go to Settings → Ports → Port Forwarding
  3. Add rule: External port 51820/UDP → Pi's local IP, port 51820

Security Hardening

  • Change default password: passwd
  • SSH key auth: copy your public key to ~/.ssh/authorized_keys
  • Install fail2ban: sudo apt install fail2ban
  • Enable UFW: sudo ufw allow ssh && sudo ufw allow 51820/udp && sudo ufw enable
  • Auto-updates: sudo apt install unattended-upgrades

Auto-Start on Boot

# WireGuard
sudo systemctl enable wg-quick@wg0

# Web app (create systemd service)
sudo tee /etc/systemd/system/rocket-shield-web.service > /dev/null <<EOF
[Unit]
Description=Rocket Shield Web App
After=network.target

[Service]
Type=simple
WorkingDirectory=/home/pi/rocket-shield-vpn
ExecStart=/usr/bin/python3 -m http.server 8080
Restart=always

[Install]
WantedBy=multi-user.target
EOF
sudo systemctl enable rocket-shield-web
💡 WireGuard is extremely lightweight — it barely uses any CPU on the Pi, even under load. It's the perfect protocol for single-board computers.

Web App Architecture

The entire web app is a single HTML file (~170KB) with embedded CSS, JavaScript, translations, and a base64-encoded logo. Zero external dependencies (except optional Google Fonts CDN).

Page System

# Navigation
showPage('command')  → shows #page-command
showPage('academy')  → shows #page-academy
showPage('launchpad')→ shows #page-launchpad
showPage('freebox')  → shows #page-freebox

# Each page is a div with class="page", toggled via .active class

CSS Theme System

Themes use CSS custom properties on [data-theme="name"] selectors. Switching themes changes the data-theme attribute on <html>, and all CSS vars cascade automatically.

i18n System

# Translation object
const i18n = { en: {...}, fr: {...}, ar: {...} }

# Helper function
function t(key) { return i18n[lang][key] || i18n['en'][key] || key; }

# HTML static elements use data-i18n attribute
<span data-i18n="navCommand">Command Center</span>

# Dynamic content uses t() in template literals
html = `<h1>${t('fb1title')}</h1>`

State Management

All state is managed with simple module-level variables:

let connected = false;      // VPN simulator connected state
let lang = 'en';            // Current language
let currentStep = 0;        // Launch Pad step
let freeboxStep = 0;        // Freebox step
let freeboxChecks = [...];  // Checklist state
let completedCards = {...}; // Academy progress
